The user can select which nanomaterials (via a list of checkboxes) are used in this item or select 'No nanomaterials are used in this item', to continue with the individual items task journey.

    1. if the product does use nanomaterials and the user has not previously added any nanomaterials to the draft they are instructed to return to the first task and add them. The status of 'This' current task, and all items returns to being 'Cannot start yet' until the nanomaterials list is completed
    2. if a user has failed to add a specific nanomaterial to their nanomatials list which is now required, they are instructed to return to the draft page and add it (via the nanomaterials Add link). The status of items which have not yet been completed return to 'Cannot start yet' until the nanomatials list is completed
  • When adding a new item: The user goes straight to the first page of the individual items task (journey)
  • When adding a new item, a corresponding entry in the Categories and formulation section appears (with 'Cannot start yet' status)
  • The last page of the items task journey asks the user if they want to complete the next item (if there is one) or return to the draft page

This logic keeps the functionality requirements for Add/Remove in the nanomaterials section to a minimum

When returning to the draft page after completing the individual item task/journey, the new item is shown appended to the items list (with a completed status). The list remains in its original order.
