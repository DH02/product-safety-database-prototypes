Add a cosmetic product - Draft

Each of the following user-flow versions provide click-through examples for adding different kinds of cosmetic products.

Clicking one of these links is the equivalent to clicking the current 'Add cosmetic products' button.

  • Add a simple product (with no multipule items or nanomaterials).
    Version 1
  • Add a product (with 3x nanomaterials).
    Version 2
  • Add a product (with with 4x nanomaterials as a multi-item kit - featuring 3x items).
    Version 3

A key technical consideration is the ability for the user to click any task already completed and redo it. On completion this will potentially update and change the status of the tasks list. If the user re-runs the first task and adds nanomaterials or a multi-item kit, this will update the task list – requiring the draft page to check for these settings by default whenever it is launched, and if the tasks for any items/nanomaterials are not yet completed the status of the following tasks are ‘Cannot start yet’. (Users are warned when rerunning tasks – that items/nanomaterials will be deleted etc, if changing their status to no/none).

For development ease, we keep tasks linear, so that they can't be started before the previous section is completed. (A future featue could be, for single products, to complete section 3 before 2, and 4 before completing any nanomaterials, etc).

The back link (at the top of the page) in the real service will return the user to the Cosmetic Products page.
