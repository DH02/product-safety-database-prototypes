Add a cosmetic product - Draft

Each of the following user-flow versions provide click-through examples for adding different kinds of cosmetic products.

Clicking one of these links is the equivalent to clicking the current 'Add cosmetic products' button.

  • Add a simple product (with no multipule items or nanomaterials).
    Version 1
  • Add a product (with 3x nanomaterials).
    Version 2
  • Add a product (with with 4x nanomaterials as a multi-item kit - featuring 3x items).
    Version 3
  • Blocked page (You cannot notify this product until you have notified the nanomaterial).
    Blocked 1
  • Blocked page (You cannot notify products containing this nanomaterial).
    Blocked 2

A key technical consideration is the ability for the user to click any task already completed and redo it. On completion this will potentially update and change the status of the tasks list. If the user re-runs the first task and adds nanomaterials or changes the product to a multi-item kit, this will update the task list – requiring the draft page to check for these settings by default whenever it is launched. (Users are warned when rerunning tasks – that items/nanomaterials will be deleted etc, if changing their status to no/none - as in flow diagram).

If re-running the first task after several nanomaterials have already been added, the 'Does the product contain Nanomaterials?' page is updated by removing the option to change the quantity. i.e, the quanity input is only offered if the product does not already have a number of nanomaterials associated with it. (The user can use the Add/Remove nanomaterials options via the tasks page).

If re-running the first task after already setting the product status to a multi-item kit, the 'Yes or No, Multi-tem kit' page is updated by removing the option to change the quantity. i.e, the quanity input is only offered if the product does not already have a number of items associated with it. (The user can use the Add/Remove items options via the tasks page).

The logic is for what happens in what tasks is prioritised for most instances by the user wanting to see the full list of tasks as early as possible (to avoid it growing or changing after the initial section sets it up). Keeping some pages out of a task also allows it to be easily updated and changed by the user again later (making the task independent).

For development ease, we keep tasks linear, so that they can't be started before the previous section is completed. Future behaviour might include the ability to leave a task halfway through and when clicking the task again on the draft page the user restarts the task where they left off? (A GDS 'In progress' tab status - pale blue - already exists).

The blocked pages/journeys require that we record the nanomaterial name and which in one scenario [blocked 1 - when the user needs to first notifiy a nanomaterial before continuing] can then be picked up again as 'Not started' (but already using the name?)

The back link (at the top of the page) in the real service will return the user to the Cosmetic Products page.
