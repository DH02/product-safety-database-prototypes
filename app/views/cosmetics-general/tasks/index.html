Tasks based user-journey:

New HTML/design page templates

Here are the HTML/design page templates for all the new pages and designs required for implimenting the first version of the new 'add a product' tasks based user-journey.

Example A:
Add a simple product - no multi-items or nanomaterials

  • Tasks list page 1 User clicks 'Add product' and lands here - seen as the user first visits the tasks list page. Their first task is 'Create the product'. In the help section is it possible on this first task to use the title: "In this first task" ?
  • Tasks list page 2 The user has completed the 'Create the product' task. The task list page now looks like this: The sub title has now updated and the main title has now changed to the product name. The header hint text has now also updated. The 'View the draft' and 'Delete the draft' links now appear.
  • Tasks list page 3 The user has completed the 'Product details' task and can now start task 4.

Example B:
Add a product - with 3x nanomaterials

  • Tasks list page 1 User clicks 'Add product' and lands here - seen as the user first visits the tasks list page. Their first task is 'Create the product'.
  • Tasks list page 2b The user has completed the 'Create the product' task. The task list page now looks like this: The sub title has now updated and the main title has now changed to the product name. The header hint text has now also updated. The 'View the draft' and 'Delete the draft' links now appear.

    In task 1 the user defined the product to have 3x nanomaterials. An extra task ('Nanomaterials') has now been inserted into task position 2, and shows 3x nanomaterial holding places (with an Add and Remove link).
  • Tasks list page 3b Each nanomaterial task has been completed. (As each one was completed, the corresponding 'Not started' tag changed to 'Completed'). The nanomaterials list now shows/uses each nanomaterial name.

    Now that all the nanomaterial tasks are complete: the user can now start task 3 ('Product details').
  • Tasks list page 4b The user has completed the 'Product details' task and can now start task 4.

Example C:
Add a product - as a multi-item kit (featuring 3x items) with 3x nanomaterials

  • Tasks list page 1 User clicks 'Add product' and lands here - seen as the user first visits the tasks list page. Their first task is 'Create the product'.
  • Tasks list page 2c The user has completed the 'Create the product' task. The task list page now looks like this: The sub title has now updated and the main title has now changed to the product name. The header hint text has now also updated. The 'View the draft' and 'Delete the draft' links now appear.

    In task 1 the user defined the product to have 3x nanomaterials and 3x items in a multi-item kit. A Nanomaterials task has now been inserted into the task list, and shows 3x nanomaterial holding places (with an Add and Remove link).

    The user can now start the next task.
  • Tasks list page 3c Each nanomaterial task has been completed. (As each one was completed, the corresponding 'Not started' tag changed to 'Completed'). The nanomaterials list now shows/uses each nanomaterial name.

    Now that all the nanomaterial tasks are complete: the user can now start the next task ('Multi-item kit').
  • Tasks list page 3d The user has completed the 'Multi-item kit' task. Each item in the 'Multi-item details' can now be started. There is a small change to the first line of the accompanying help text (differing from the single product version).
  • Tasks list page 4d The user has completed the 'Multi-item details' tasks. (As each item task was completed, the corresponding 'Not started' tag changed to 'Completed'). The items list now shows/uses each item name.

    The user can now start the final task.

Delete draft pages

  • New page: Delete draft
    This is the "Delete draft" page. Yes or No. (Error message would state: "Select yes if you want to delete this draft"). The Cancel link is a hyperlink linking back to the draft page. The hint text includes the product name. Choosing No, returns the user back to the same product draft page. Yes, deletes the draft and the user goes to ...
  • New page: Delete success
    This page confirms the deletion. A hyperlink provides a direct route back to the Cosmetic products page. There is no Back link on this page (and a 'govuk-main-wrapper--auto-spacing' class is added to the <main> element if possible).

Nanomaterials

  • New page: What is the nanomaterial INCI name
    This is the new page to start the individual nanomaterial tasks. On this page we Save and continue (so that the name is instantly assigned to the nanomaterial instance in the task list, even if the task is not completed) and the new name is used in the task list (but the task status tag remains as 'Not started'). If the user does the task again, or restarts the task - for a nanomaterial - this input 'value' will be the name the user provided (and is obviously editable). Only completing the task does the task become flagged as Complete. The form error message will be "Enter the nanomaterial INCI name" - with INCI inside plain abbr tags if possible.
  • New feature: Add (a nanomaterial)
    The Add link simply goes to the first page of the new nanomaterials task ("What is the nanomaterial INCI name?"), and on completion adds the nanomaterial to top of the nanomaterial list as 'Completed'.
  • New page: Remove (nanomaterials)
    "Select which nanomaterial(s) to remove" - This is the new page to remove one or more nanomaterials: the user selects from a checkbox list of the nanomaterial names (even if they are still "Nanomaterial 1, 2, etc" and a 'Delete and continue' button returns the user to the task list page with the list updated. (No standalone confirmation page required and a green button is fine). The page includes hint text advising that if all nanomaterials are deleted they must complete the first task again to add nanomaterials. The Cancel link returns the user back to the Tasks list page.

Complete product / Multi-item details

  • New page: What is the item called?
    This is the new page that only appears at the start of the penultimate task (Multi-item details) if the product is a multi-item kit (as the user begins the individual tasks of defining each item). On this page we Save and continue (so that the name is instantly assigned to the item instance in the task list, even if the task is not completed) and the new name is used in the task list (but the status tag remains 'Not started').
  • New page: Select which nanomaterials are included in the item
    This is the next new page in the flow (after 'What is the item called?', above) and only appears if the product has nanomaterials defined. A 'Cancel' link - next to the Continue button, is a hyperlink linking back to the task list page.
  • Note: the next page in the flow ("Is the product intended to be rinsed off or left on?") is not a new page but it also only appears if the product has nanomaterials defined.
