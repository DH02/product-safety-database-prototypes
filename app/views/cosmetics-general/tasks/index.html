Tasks based user-journey:

New HTML/design page templates

Here are the HTML/design page templates for all the new pages and designs required for implimenting the first version of the new 'add a product' tasks based user-journey.

Example A:
Add a simple product - no multi-items or nanomaterials

  • Tasks list page 1 User clicks 'Add product' and lands here - seen as the user first visits the tasks list page. Their first task is 'Create the product'.
  • Tasks list page 2 The user has completed the 'Create the product' task. The task list page now looks like this: The sub title has now updated and the main title has now changed to the product name. The header hint text has now also updated. The 'View draft' link now appears.
  • Tasks list page 3 The user has completed the 'Product details' task and can now start task 4.

Example B:
Add a product - with 3x nanomaterials

  • Tasks list page 1 User clicks 'Add product' and lands here - seen as the user first visits the tasks list page. Their first task is 'Create the product'.
  • Tasks list page 2b The user has completed the 'Create the product' task. The task list page now looks like this: The sub title has now updated and the main title has now changed to the product name. The header hint text has now also updated. The 'View draft' link now appears.

    In task 1 the user defined the product to have 3x nanomaterials. An extra task ('Nanomaterials') has now been inserted into task position 2, and shows 3x nanomaterial holding places (with an Add and Remove link).
  • Tasks list page 3b Each nanomaterial task has been completed. (As each one was completed, the corresponding 'Not started' tag changed to 'Completed'). The nanomaterials list now shows/uses each nanomaterial name.

    Now that all the nanomaterial tasks are complete: the user can now start task 3 ('Product details').
  • Tasks list page 4b The user has completed the 'Product details' task and can now start task 4.

Example C:
Add a product - as a multi-item kit (featuring 3x items) with 3x nanomaterials

  • Tasks list page 1 User clicks 'Add product' and lands here - seen as the user first visits the tasks list page. Their first task is 'Create the product'.
  • Tasks list page 2c The user has completed the 'Create the product' task. The task list page now looks like this: The sub title has now updated and the main title has now changed to the product name. The header hint text has now also updated. The 'View draft' link now appears.

    In task 1 the user defined the product to have 3x nanomaterials and 3x items in a multi-item kit. A Nanomaterials task has now been inserted into the task list, and shows 3x nanomaterial holding places (with an Add and Remove link).

    A multi-kit task and 3x items tasks replace the 'Product details' task, and have now also been inserted into the list. The list now has 5 tasks.

    The user can now start the next task.
  • Tasks list page 3c Each nanomaterial task has been completed. (As each one was completed, the corresponding 'Not started' tag changed to 'Completed'). The nanomaterials list now shows/uses each nanomaterial name.

    Now that all the nanomaterial tasks are complete: the user can now start the next task ('Multi-item kit').
  • Tasks list page 3d The user has completed the 'Multi-item kit' task. Each item in the 'Multi-item details' can now be started.
  • Tasks list page 4d The user has completed the 'Multi-item details' tasks. (As each item task was completed, the corresponding 'Not started' tag changed to 'Completed'). The items list now shows/uses each item name.

    The user can now start the final task.

This functionality has been removed from the designs

Does it contain nanomaterials?

  • New page: Does it contain nanomaterials?
    This is a revised version of the nanomaterials question now in task 1. As the user completes task 1 they are asked here about nanomaterials and, if yes, how many. But they are also able to choose 'Don't know'. (This was agreed with Nasir to avoid users dropping out of the task if they didn't know the nanomaterials answer.
  • Task list page, showing nanomaterials question In task 1, where the user is asked "Does the product contain nanomaterials?" And they can answer Yes (+ quantity), No or Don't know. If they answer 'Don't know' the user continues with the standard user-flow, but when they are eventually returned back to the Task list page, there is task 1 (completed) and a new task 2 displaying: "About nanomaterials". This new task 2 gives the user a chance to source the answers. But they must do task 2 before they can continue).
  • New page: Does it contain nanomaterials? (short version)
    From task 2 (the About nanomaterials task) the user goes to this page (with only two options and a Cancel link). Afterwards, the user is returned back to the task list page as normal and as if they had answered the nanomaterials question in the original task 1 (making the task list look like this if they had entered 3x, for example).
